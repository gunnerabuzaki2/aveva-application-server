Q2: In the Security Model, what is the relationship between Security Groups, Roles, and Users?

Click to reveal answer

Security Groups contain AutomationObjects. Roles are granted Operational Permissions onto Security Groups. Users belong to one or more Roles and receive the sum of all permissions from their assigned Roles.

Q3: What is the difference between "Can Modify Operate Attributes" and "Can Modify Configure Attributes"?

Click to reveal answer

"Can Modify Operate Attributes" allows daily operational tasks like changing setpoints, outputs, and control modes. "Can Modify Configure Attributes" allows major configuration changes and requires the object to be set to Off scan first.

Q4: Why did the Plant Maintenance user receive an "Access Denied" message when trying to open the IDE?

Click to reveal answer

The Plant Maintenance 1 role was not granted "Can Start the IDE" permission (under IDE Permissions in General permissions). This role only had SMC Permissions and Operational permissions (Tune and Configure) for the production lines.

Q5: What happens to all new objects created in a Galaxy with respect to Security Groups?

Click to reveal answer

All new objects belong to the Default Security Group by default. This group cannot be removed. Objects must be manually moved to other security groups as needed.

Q6: What is the Login time setting used for in OS Group based authentication, and why did the lab set it to 3000?

Click to reveal answer

Login time (in milliseconds) is the timeout period for validating user membership against OS groups. After the timeout, the system defaults to the local cache. The lab set it to 3000 ms to allow for a more seamless experience when testing different users, especially in intermittent or slow network scenarios.

Q7: What is the Plant Supervisors 1 role allowed to do that Plant Operators 1 is not?

Click to reveal answer

Plant Supervisors 1 has access to all three operational areas (Line1, Line2, ControlSystem) and has Can Verify Writes permission. Plant Operators 1 only has access to Line1 (Can Acknowledge Alarms + Can Modify Operate Attributes) and no Verify Writes permission.

Q8: After enabling security and assigning new Security Groups, what must happen to deployed objects before they function correctly?

Click to reveal answer

Objects must be redeployed because their SecurityGroup attribute was modified when they were moved between security groups. The IDE will show objects needing deployment, and this will be addressed in Lab 17.

Q9: Can you modify the General permissions for the Administrator role? Why or why not?

Click to reveal answer

No, you cannot modify the General permissions for the Administrator role. The system enforces that the Administrator always has full General permissions to prevent accidental lockout from critical system functions.

Q10: What security limitation should be aware of regarding password storage in Galaxy authentication mode?

Click to reveal answer

Passwords are encrypted but stored in a database that is accessible. The system is not designed to stop determined programmers from accessing it. For higher-level security, use OS-based authentication with Microsoft tools, password timeout, and electronic signature authentication.
